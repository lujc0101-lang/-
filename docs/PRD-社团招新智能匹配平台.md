| 属性 | 内容 |

## 1. 背景与目标

### 1.1 背景

高校社团招新存在：信息不对称、报名流程分散、新生难以快速找到兴趣契合的社团等问题。v2.0 在「发现—匹配—报名」主线之上，增加 **社区化信息流**，用帖子承载招新宣传、活动与讨论，并统一 **内容审核** 与 **底部导航** 体验。

### 1.2 产品目标

- **新生 / 访客**：浏览 **社区帖子**（可搜索、按类型筛选）→ 进入 **社团区** 发现社团与 **问卷/对话匹配** → 社团详情 → **一键报名**（含从招新帖进入）；登录后可发帖（先审后发）、点赞、评论、收藏社团、查看「我的」与通知。
- **社长 / 招新负责人**：维护社团资料与报名表、处理报名、查看社团数据简版；可发布带社团关联的帖子（同样先审后发，由管理员审核）。
- **平台管理员**：审核 **社团入驻**、审核 **社区帖子**（通过 / 拒绝 / 置顶）、生成社长邀请码、查看运营看板。

### 1.3 成功指标（建议，可按学期迭代）

| 指标 | 说明 | 备注 |
|------|------|------|
| 报名完成率 | 进入报名表单的用户中完成提交占比 | P0 需可统计 |
| 推荐点击率 | 推荐列表卡片点击占比 | P0 需可统计 |
| 内容审核时效 | 帖子从待审到处理的时间分布 | 社区上线后建议跟踪 |
| 社长处理时效 | 报名状态首次更新时间分布 | P1 |
| 用户满意度 | 学期末问卷 NPS/CSAT | 运营向 |

## 2. 用户与角色

### 2.1 角色定义

| 角色 | 标识 | 权限概要 |
|------|------|----------|
| 访客 | 未登录 | 浏览**已通过审核的**社区帖子、社团目录与详情；可完成匹配问卷/对话（部分能力可匿名）；不可提交报名、不可发帖/评论/点赞（或引导登录） |
| 新生 | `student` | 问卷/对话匹配、**收藏社团**、提交报名、查看「我的报名」；维护个人画像；**发布帖子**（默认待审核）、点赞、评论 |
| 社长 | `leader` | 绑定一个或多个社团；编辑社团资料与报名表、处理报名、查看社团数据；**发帖**（待审核）；工作台入口 |
| 管理员 | `admin` | 社团审核、**帖子审核**（先审后发）、邀请码、运营看板 |

### 2.2 假设

- 同一用户可**仅为一种业务角色**（一期简化：`student` 与 `leader` 分身账号二选一；兼任可用两账号或二期合并身份）。
- 社长绑定社团需**可验证**（`admin` 审核 或 邀请码绑定）。

### 2.3 v2.0 相对 v1.0 的能力调整（摘要）

| v1.0 设计 | v2.0 现状 |
|-----------|-----------|
| 首页 = 发现 + 公告 + 专场入口 | **首页 = 社区信息流**；公告/专场**不设独立模块**，由**置顶帖、活动类帖子**等承接 |
| 独立「对比」页 | **已移除**；收藏仍保留，可从收藏进详情/报名 |
| 独立「招新专场 / 展位 / 联办活动」数据模型与页面 | **已移除**；活动信息用 **ACTIVITY 类帖子** 表达 |
| 全站公告后台 | **已移除**；重要信息用 **置顶 + 招新/活动帖** |
| 匹配入口在顶栏 `/match` | **问卷匹配**、**对话匹配**归入 **社团区** `/clubs/match`、`/clubs/match/nl`；旧路径重定向 |
| 无 UGC 帖子 | **新增** Post 模型；**先审后发**（`PENDING` → `APPROVED` / `REJECTED`） |

## 3. 范围与分期（与实现对齐）

### 3.1 已实现核心能力（验收以当前代码为准）

1. **社区**：帖子列表（搜索正文与标签、按类型筛选）、帖子详情、点赞、评论、发帖页、**招新帖关联社团时提供「一键报名」**。
2. **内容安全**：新帖默认 **待审核**；仅 **已通过** 帖子对公众可见；作者与管理员可查看本人/待审队列。
3. **社团目录**：分类、搜索、排序、招新筛选；详情（含 **联系方式结构化展示**，非 JSON 裸显）。
4. **智能匹配**：固定问卷 → 规则打分 → Top N + 可解释理由；**对话式匹配**（规则/关键词抽取）与问卷共用打分逻辑。
5. **报名**：通用字段 + 社团自定义题；防重复（userId + clubId + seasonId）；状态机与时间轴。
6. **账号**：注册登录、`student` / `leader` / `admin`；社长工作台；管理端（社团、**帖子审核**、邀请码、看板）。
7. **通知**：报名相关、帖子审核结果、评论等（站内）。
8. **埋点**：列表/详情/匹配/报名等事件。
9. **导航**：顶栏（社区、社团、发布）+ **底栏**（社区｜社团｜发布｜我的）。

### 3.2 可选后续增强（未在本文档逐条绑定实现）

- 邮件通知、WebSocket 实时通知、更细的内容风控、图片上传至对象存储、生产环境 PostgreSQL 与部署流水线等。

## 5. 功能需求详述

以下每条含 **描述**、**规则**、**验收标准（AC）**。

### 5.1 社区信息流（P0 / v2 核心）

**描述**  
首页展示 **已通过审核** 的帖子，支持 **搜索**（标题、正文、标签）、**类型筛选**（招新 / 活动 / 讨论）。置顶帖优先展示。

**规则**

- 帖子类型：`RECRUIT`（招新）、`ACTIVITY`（活动）、`DISCUSS`（讨论）。
- 默认仅 `APPROVED` 对所有人可见；`PENDING` 仅作者与管理员可见（用于预览或审核）。
- 帖子可关联 `clubId`；列表/详情展示社团名称（若有）。
- 埋点：如 `community_feed_view`。

**验收标准**

- AC1：无登录可浏览已通过帖子列表与详情（除需登录的互动外）。
- AC2：搜索与类型筛选结果正确；空状态有引导（如去发布）。
- AC3：置顶帖在列表中优先展示。

---

### 5.2 帖子详情与互动（P0）

**描述**  
单帖正文、图片、标签、作者与关联社团；**点赞**、**评论**（登录）；招新帖且关联已发布社团时展示 **一键报名** 与 **查看社团**。

**规则**

- 评论写入后通知帖子作者（站内通知，实现可异步/容错）。
- 仅 `APPROVED` 帖子允许点赞与评论。

**验收标准**

- AC1：未登录用户点击赞/评时引导登录并回跳。
- AC2：招新帖「一键报名」进入 `/apply/:clubId`。

---

### 5.3 发布与先审后发（P0）

**描述**  
登录用户填写类型、标题（可选）、正文、标签、可选关联社团；提交后状态为 **待审核**；管理员在后台 **通过 / 拒绝**（可填拒绝说明）；通过后可 **置顶**。

**规则**

- 拒绝后通知作者；通过后在社区公开展示。
- 图片 URL（若有）在创建接口中校验格式；演示环境可使用站内静态资源。

**验收标准**

- AC1：新帖默认 `PENDING`，不出现在公开列表。
- AC2：管理员通过后立即可在首页看到；拒绝后作者收到通知。

---

### 5.4 社团目录（P0）

（同 v1：已发布社团、搜索、排序、招新筛选、卡片字段等。）

**验收标准**

- AC1～AC3：与 v1 一致；封面图建议使用**站内可访问资源**，避免外链不可用导致裂图。

---

### 5.5 社团详情（P0）

（同 v1 字段与招新状态、PV、报名按钮等。）

**补充（v2）**

- **联系方式**：以 **标签 + 文案** 展示（如邮箱、QQ 群、微信群关键词等），**禁止**以原始 JSON 块展示；字段键名到中文标签可配置映射。

**验收标准**

- AC1～AC3：同 v1；联系方式符合结构化 UI 要求。

---

### 5.6 智能匹配 — 问卷（P0）

**描述**  
入口位于 **社团区** `/clubs/match`。固定问卷 → 可配置权重规则 → Top N + **匹配理由**。登录可选保存到 `MatchProfile`。

**验收标准**

- AC1～AC4：同 v1「5.3 智能匹配」。

---

### 5.7 对话式匹配（P2，已实现）

**描述**  
入口位于 **社团区** `/clubs/match/nl`。自然语言 → 规则/关键词抽取结构化偏好 → **与问卷同一打分引擎**；失败时引导使用问卷。

**验收标准**

- AC1～AC2：同 v1「5.8 对话式匹配」。

---

### 5.8 收藏（P0，无对比页）

**描述**  
登录用户可收藏社团；**不提供** 2～3 社「对比」独立页；可从收藏跳转详情与报名。

**验收标准**

- AC1：收藏持久化，列表展示正确。

---

### 5.9 报名（P0）

（同 v1：通用字段、自定义题、防重复、状态机、时间轴。）

**验收标准**

- AC1～AC3：同 v1「5.5 报名」。

---

### 5.10 通知（P1）

**描述**  
报名状态变更、帖子审核结果、帖子评论等；站内列表与未读；用户设置中可关闭部分提醒（若已实现）。

**验收标准**

- AC1：通知含可跳转链接（帖子、报名、社团等）。

---

### 5.11 运营看板（P2）

**描述**  
管理员查看曝光、漏斗、社团排行等（具体指标以实现对齐）。

**验收标准**

- AC1：仅 `admin` 可访问。

---

### 5.12 ~~招新专场 / 联办活动 / 全站公告独立模块~~（v1 已规划，v2 已下线）

**说明**  
不再提供独立「专场页」「展位表」「联办活动实体」「公告后台」。等效运营动作通过 **社区帖子**（活动类、置顶招新类）完成。

---

### 5.13 社长工作台（P0）

（同 v1：资料、报名表、报名列表、简版数据。）

---

### 5.14 管理后台（P0）

- 社团入驻审核：`pending_review` → `published` / `rejected`。
- **帖子审核**：待审列表、通过/拒绝、置顶（对已通过的帖子）。
- 生成社长邀请码。

## 6. 数据模型（逻辑层，与 v2 实现对齐）

### 6.1 核心实体

**User**  
`id`, `role`, `email`, `passwordHash`, `name`, `studentId?`, `profile`, `notifyDeadlineReminders`, …

**Club**  
`id`, `name`（unique）, `slogan`, `category`, `tags`, `description`, `coverUrl`, `gallery`, `activityTypes`, `intensity`, `beginnerFriendly`, `recruitStart`, `recruitEnd`, `recruitQuotaNote`, `applyNote`, `contact`（JSON）, `customForm`, `leaderUserId`, `status`, `seasonId`, …

**Post**（v2 新增）  
`id`, `authorId`, `type`（RECRUIT | ACTIVITY | DISCUSS）, `title?`, `content`, `images`（JSON 数组）, `clubId?`, `tags`（JSON）, `status`（PENDING | APPROVED | REJECTED）, `pinned`, `rejectNote?`, `reviewedAt?`, `reviewerId?`, …

**PostLike**、**PostFavorite**、**PostComment** — 与 Post、User 关联。

**Application**  
同 v1：`userId`, `clubId`, `seasonId`, `commonAnswers`, `customAnswers`, `status`, `timeline`, `clubSnapshot?`, …

**Favorite**（社团收藏）  
`userId`, `clubId`，unique(userId, clubId)

**MatchProfile**  
`userId`, `answers`, `nlSource?`, …

**Notification**  
`userId`, `type`, `payload`（JSON）, `readAt`, …

**AnalyticsEvent**  
`type`, `clubId?`, `userId?`, `sessionId`, `ts`, `meta?`

**InviteCode**  
社长绑定与邀请

### 6.2 已移除实体（相对 v1 PRD，代码中不再使用）

- `Announcement`（全站公告）
- `RecruitmentEvent`、`ClubEventBooth`（招新专场与展位）
- `JointEvent`（联办活动）

### 6.3 索引建议

- `Post(status, pinned, createdAt)`、`Post(clubId)`、`Post(type, status)`  
- `Application(userId, clubId, seasonId)` unique、`Application(clubId, status)`

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首屏可接受范围内加载；列表限制条数，避免一次拉取过大 |

| 无障碍 | 主要交互控件具备可访问名称（建议） |
| 国际化 | P0 中文即可 |

## 9. 边界与异常

- 社团下架：详情 404；已报名可在「我的报名」查看（快照策略同 v1）。
- 帖子删除/拒绝：公开列表不可见；作者可收到通知。
- 并发报名：唯一约束兜底。
